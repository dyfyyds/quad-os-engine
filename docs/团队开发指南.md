# Quad-OS 团队开发指南

本平台已完成**前端脚手架**（综合 OS 模拟平台：总览大屏 + 四核心联动 + 实时监控/事件/告警/设置，mock 数据驱动）。本指南指导**人类团队**把各核心从 mock 替换为真实算法逻辑，多人并行开发。

## 一、运行项目

```bash
cd frontend
npm install
npm run dev        # http://localhost:5173，纯前端 + mock，无需后端
```

后端（第一阶段已完成的算法引擎，可选）：
```bash
cd backend && pip install -r requirements.txt && uvicorn main:app --port 8080
```

## 二、模块边界与负责人划分（可并行）

每条工作线相互独立，只通过中央 store 的字段契约（见 [接口契约.md](接口契约.md)）交互，互不阻塞。

| # | 工作线 | 负责文件 | 任务 |
|---|---|---|---|
| 1 | **处理机调度核心** | `views/cores/ProcessorCore.vue`、driver 中处理机段 | 接入 `/api/scheduling/run`，实现真实 FCFS/SJF/HRRN/优先级/RR；PCB 表与甘特图改用真实 steps |
| 2 | **存储管理核心** | `views/cores/MemoryCore.vue`、driver 存储段 | 接入 `/api/paging/run` 与 `/translate`，真实 FIFO/LRU/OPT/CLOCK 缺页统计与地址转换 |
| 3 | **进程与资源核心** | `views/cores/ResourceCore.vue`、driver 资源段 | 接入 `/api/banker/*` 与 `/api/sync/run`，真实银行家安全序列与 PV 同步 |
| 4 | **设备管理核心** | `views/cores/DeviceCore.vue`、driver 设备段 | 接入 `/api/disk/run`，真实移臂调度与磁头轨迹 |
| 5 | **总览大屏 / 平台** | `views/Dashboard.vue`、`RealtimeMonitor/EventQuery/AlarmCenter`、`store/os.js` | 全局指标聚合、事件流规则、告警阈值、联动编排 |
| 6 | **可视化组件** | `components/widgets/*`、`components/viz/*` | 仪表盘/趋势/甘特/磁头/页框/矩阵 组件增强 |

> 公共契约文件 `store/os.js` 由工作线 5 统筹，其他线**只读不改字段名**，需扩展字段先在此协商。

## 三、mock → real 替换流程

1. 阅读 [接口契约.md](接口契约.md)，确认本核心负责的 store 字段与对应 API 端点。
2. 在 `frontend/src/mock/driver.js` 的 `tick()` 中找到本核心的 mock 段（含 `TODO(team)` 注释）。
3. 把 mock 变更替换为：按虚拟时钟调用 `api.xxx()`（`frontend/src/api/client.js` 已封装全部后端接口），将返回写回 store 对应字段。
4. 页面**无需改动**（已读 store）；如需更精细的分步动画，参考第一阶段算法页 `views/JobScheduling.vue` 等对 `steps[]` 的用法（保留在仓库）。
5. 后端真实算法已在 `backend/app/engines/*` 实现并通过 40 个 pytest 用例，可直接复用。

## 四、分支策略

- `main`：稳定脚手架，受保护。
- 每条工作线一分支：`feat/core-processor`、`feat/core-memory`、`feat/core-resource`、`feat/core-device`、`feat/platform`。
- 提交规范：`feat(processor): 接入真实调度引擎`；PR 需通过 `npm run build` 与（如涉及后端）`pytest`。
- 因各线只读独立 store 字段，合并冲突主要集中在 `store/os.js`（由工作线 5 仲裁）。

## 五、验收标准（每条工作线）

- `npm run build` 通过、无 console 报错。
- 点击顶部「运行 / 单步」，本核心页数据来自**真实算法**而非 mock。
- 总览大屏对应指标随之联动更新。
- 与教材已知答案一致（如 SCAN 总寻道 299/331、FIFO 缺页 15 次等，见 `backend/tests/`）。
