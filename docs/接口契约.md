# Quad-OS 接口契约

本文件定义平台的**唯一数据契约** —— 中央 OS 状态 store 的字段结构，以及未来各核心团队需实现的**后端 API 端点**。

> 核心原则：**所有页面只读中央 store**（`frontend/src/store/os.js`）。无论数据来自 mock 驱动还是真实算法引擎，只要保持下列字段结构不变，前端页面零改动。这是 mock→real 平滑替换的关键。

---

## 一、中央 OS 状态结构（`store/os.js`）

```ts
{
  clock: number            // 虚拟时钟
  running: boolean         // 是否运行中
  speed: number            // 速度倍率 0.5/1/2/4

  // 处理机核心 ——————————————————————————
  processes: Array<{
    pid: number; name: string;
    state: '运行'|'就绪'|'阻塞'|'完成';
    arrival: number; burst: number; ran: number; priority: number;
  }>
  gantt: Array<{ 作业: string; 开始: number; 结束: number }>
  nextPid: number

  // 存储核心 ————————————————————————————
  memory: {
    capacity: number;
    frames: Array<number|null>;            // 各物理块当前页号，null=空
    pageTable: Array<{ 页号:number; 标志:0|1; 主存块号:number|null }>;
    refString: number[]; refPtr: number;
    faults: number; hits: number;
  }

  // 资源核心（银行家）————————————————————
  resources: {
    types: string[];
    available: number[];
    max: number[][]; allocation: number[][]; need: number[][];
    safeSeq: string[]; deadlock: boolean;
  }

  // 进程同步（PV）————————————————————————
  sync: { capacity:number; s1:number; s2:number; buffer:number;
          produced:number; consumed:number;
          prodBlocked:string[]; consBlocked:string[] }

  // 设备核心（磁盘）——————————————————————
  disk: { trackCount:number; head:number; queue:number[];
          path:number[]; totalSeek:number; served:number }

  // 全局 ————————————————————————————————
  events: Array<{ ts:number; type:string; core:string; level:'info'|'warning'|'danger'; desc:string }>
  metrics: { cpuUtil:number; memUtil:number; throughput:number; faultRate:number;
             avgTurnaround:number; readyLen:number; blockedLen:number;
             diskQueueLen:number; completed:number }
  history: { labels:number[]; cpu:number[]; mem:number[]; fault:number[]; throughput:number[] }
  config: { quantum:number; memFrames:number; resTotal:number; trackCount:number;
            schedAlgo:string; pageAlgo:string; diskAlgo:string; clockSpeed:number }
}
```

`core` 取值：`processor` | `memory` | `resource` | `device` | `system`。

## 二、未来后端 API 端点契约（mock→real 替换点）

第一阶段已实现的算法引擎位于 `backend/app/engines/*`，并通过 `backend/app/api/*` 暴露为 REST 接口（见 `http://localhost:8080/docs`）。团队接入真实逻辑时，把 `frontend/src/mock/driver.js` 的 `tick()` 改为按虚拟时钟调用这些接口，并把返回结果写回 store 对应字段。

| 核心 | 端点 | 请求 | 写回 store 字段 |
|---|---|---|---|
| 处理机 | `POST /api/scheduling/run` | `{algorithm, jobs, time_quantum}` | `gantt`、`processes[].state`、`metrics.avgTurnaround` |
| 存储 | `POST /api/paging/run` | `{algorithm, reference_string, frames}` | `memory.frames`、`memory.faults/hits`、`metrics.faultRate` |
| 存储 | `POST /api/paging/translate` | `{page_table, instructions, block_size}` | `memory.pageTable` 转换结果 |
| 资源 | `POST /api/banker/safety` | `{available, max, allocation}` | `resources.safeSeq`、`resources.deadlock` |
| 资源 | `POST /api/banker/request` | `{available, max, allocation, pid, request}` | `resources.*` |
| 同步 | `POST /api/sync/run` | `{operations, buffer_size}` | `sync.*` |
| 设备 | `POST /api/disk/run` | `{algorithm, requests, head, disk_size, direction}` | `disk.path/head/totalSeek` |

所有接口返回统一的 `SimulationTrace`：`{module, algorithm, input_echo, steps[], metrics, final_state}`（定义见 `backend/app/schemas/common.py`）。各核心页可改用 `steps[]` 做精细分步动画（第一阶段算法页 `frontend/src/views/JobScheduling.vue` 等已演示该用法，保留备用）。
