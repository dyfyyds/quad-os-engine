# 数字孪生架构（Quad-OS Digital Twin）---改进方向

把「正在运行的虚拟 OS」镜像成一个可实时观察、可多端渲染、未来可接真实遥测的数字孪生。
本平台已实现 2D / 3D 两套孪生视图，架构按「**渲染无关 + 数据源可替换**」设计，便于团队扩展。

## 三层结构

```
            ┌───────────── 数据源（可替换） ─────────────┐
            │  Mock 驱动(现)   真实引擎(待接)   外部遥测(预留) │
            └──────────────────────┬─────────────────────┘
                                   ▼  写入
                       中央 OS 状态 store（Pinia, store/os.js）
                                   ▼  派生
                 世界模型 useOsWorld()（twin/world.js，渲染无关）
                                   ▼  消费
            ┌──────────────┬───────────────┬──────────────┐
            ▼              ▼               ▼              ▼
        2D SVG 孪生     3D Three.js 孪生   （未来）大屏/AR   外部系统
       DigitalTwin.vue   Twin3D.vue
```

## 1. 数据源层（可替换）

- **现状**：`mock/driver.js` 按虚拟时钟产出连贯的 OS 运行叙事，写入中央 store。
- **接真实引擎**：把 driver 的 `tick()` 内部替换为对 `backend/app/engines/*` 的真实编排（作业→分页→银行家→磁盘），**保持写入 store 的字段契约不变**。
- **接外部遥测**：实现一个新的「源」，订阅外部真实 OS/设备的指标流，按同样契约写入 store。
- 三种源对上层**完全透明** —— 世界模型与所有孪生视图无需改动。

## 2. 世界模型层（渲染无关，核心接缝）

`twin/world.js` 的 `useOsWorld()` 把 store 映射为一个**与渲染技术无关**的「OS 世界」快照：
四核心实体（标题/指标/负载 bar/健康度/活动态）+ 调度内核 + 队列 + 事件。
任何渲染器（2D/3D/大屏/AR）或外部系统都统一消费此模型，互不耦合。

## 3. 渲染层（多端，已实现 2D + 3D）

| 视图 | 文件 | 说明 |
|---|---|---|
| 2D 孪生 | `views/DigitalTwin.vue` | SVG 调度内核 + 四核心节点 + 联动通道，轻量、稳定回退 |
| 3D 孪生 | `views/Twin3D.vue` | Three.js 立体机房：核心塔（高度=负载）+ 数据流光线 + OrbitControls + 浮动数据标签（参考 ThreeMaps 思路） |

页面顶部 **3D / 2D 切换**；WebGL 不可用时 3D 自动回退提示，2D 始终可用。

## 如何扩展

- **加一个渲染器**（如 ECharts GL 大屏、AR）：新建组件，调用 `useOsWorld()` 读 `world.value` 渲染即可，无需碰数据源与 store。
- **加一类实体**（如网络/文件系统核心）：在 `store/os.js` 加字段 → 在 `world.js` 映射 → 各视图按需读取。
- **双向控制**：孪生→系统的反向指令已具雏形（顶部运行控制条 `RunControlBar` + `SystemSettings` 参数）。接真实引擎后，可在此通道注入「what-if / 控制」指令。
- **时间回放**：可在数据源层增加世界快照环形缓冲，为孪生页加回放滚动条（当前为实时态，回放为后续增强点）。
